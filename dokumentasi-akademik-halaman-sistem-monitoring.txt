2.1.2 Implementasi Antarmuka Sistem Monitoring Coverage Jaringan

Implementasi antarmuka Sistem Monitoring Coverage Jaringan pada platform Wikarta dirancang untuk mendukung proses monitoring, pengelolaan data infrastruktur, audit operasional, dan pelaporan secara terintegrasi. Seluruh halaman dibangun berbasis web dengan pendekatan modular agar pengguna dapat mengakses fungsi sistem sesuai peran (super admin, admin, teknisi).

Secara umum, antarmuka sistem terdiri dari halaman publik (sebelum autentikasi), halaman autentikasi, halaman operasional inti (dashboard, peta, jaringan), halaman tata kelola (wilayah, pengguna, peran), halaman laporan, dan halaman pengaturan akun.

A. Halaman Utama
Halaman utama merupakan tampilan awal sistem yang berfungsi sebagai gerbang informasi dan navigasi awal pengguna. Halaman ini menampilkan ringkasan fungsi sistem sebagai pusat kendali jaringan fiber, meliputi manajemen OLT, ODC, ODP, ONT, visualisasi topologi, dan aktivitas operasional lapangan.

Pada halaman utama terdapat komponen sebagai berikut:
a) Header Navigasi, terdiri dari tombol Masuk dan Daftar bagi pengguna yang belum login, serta tombol Masuk Dashboard bagi pengguna yang sudah login.
b) Informasi Utama Sistem, yang menjelaskan platform sebagai pusat kendali jaringan dan sinkronisasi data lapangan.
c) Fitur Unggulan, meliputi:
   - Monitoring status node secara real-time
   - Pemetaan topologi berbasis wilayah
   - Audit trail perubahan data
d) Tombol Aksi:
   - Mulai dari Login
   - Lihat Peta Jaringan
e) Modul Sistem, dikelompokkan ke tiga bagian:
   - Jaringan
   - Operasional
   - Tata Kelola
f) Workflow Ringkas Operasional, yang menjelaskan alur inventaris -> monitoring -> operasional.

G. Halaman Peta Jaringan

3. Panduan Pengambilan Screenshot Dokumen

3.1 Ketentuan Teknis Screenshot
Agar hasil dokumentasi konsisten dan dapat dinilai secara akademik, screenshot disarankan mengikuti ketentuan berikut:
a) Resolusi minimal 1366 x 768 (desktop).
b) Gunakan browser/viewport penuh, tanpa crop berlebihan.
c) Pastikan nama halaman dan elemen utama terlihat jelas.
d) Untuk halaman tabel, tampilkan header tabel dan minimal 3 baris data.
e) Untuk halaman map, tampilkan layer aktif yang relevan dan legenda.
f) Gunakan penomoran gambar berurutan sesuai urutan pembahasan.

3.3 Arahan Narasi per Screenshot
Untuk setiap gambar, gunakan format penjelasan berikut:
a) Tujuan halaman: menjelaskan fungsi halaman dalam alur sistem.
b) Komponen utama: menyebut elemen penting yang terlihat pada gambar.
c) Nilai operasional: menjelaskan manfaat halaman bagi monitoring dan pengelolaan jaringan.

Contoh format narasi:
"Gambar 22 menampilkan Halaman ODP Create yang digunakan untuk menambahkan node ODP baru ke sistem. Pada halaman ini, pengguna dapat memilih parent device, mengisi kapasitas, dan menentukan koordinat lokasi. Fitur visual parent marker dan garis koneksi membantu pengguna memastikan relasi topologi agar input data tidak ambigu."

3.4 Catatan Penyusunan Bab Laporan
Jika dokumentasi ini dimasukkan ke laporan kerja praktik/skripsi, disarankan menggunakan susunan:
- 2.1 Gambaran Umum Sistem

Dengan susunan tersebut, pembahasan antarmuka akan lebih sistematis, mudah ditelusuri, dan sesuai format akademik.
